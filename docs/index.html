Core Features

核心能力

不仅是聊天机器人——VoltaireAI 能理解你的网页，并在上面执行操作

🧠

意图理解

LLM 自动判断用户想"问问题"还是"做操作"，走不同的处理管线。操作类请求生成可执行代码，问题类请求基于知识库回答。

📚

知识库增强

每次对话自动检索页面元素、操作步骤、文档资料。ChromaDB 向量搜索 + LLM 语义筛选，精准匹配用户意图。

⚡

代码生成 & 执行

AI 生成分步骤的 JavaScript 操作代码，前端通过 Testing Library 安全沙箱直接执行。多步骤可逐步执行，失败自动重试。

🔄

失败自修复

代码执行出错时，错误信息自动反馈给 LLM，根据错误上下文重新生成修正代码，无需用户干预。

📡

SSE 流式推送

用户可实时看到 AI 的思考过程："分析意图 → 选择Agent → 查询知识库 → 生成回复"，不再是黑盒等待。

🎛️

配置驱动

所有 Agent 提示词、规则、模型源均通过配置文件管理。管理后台可视化管理，修改即时生效，无需重启。

Getting Started

三步集成

从零到拥有 AI 助手，只需要三步

1

启动后端服务

安装依赖并启动 FastAPI 服务，默认运行在 localhost:8001

# 安装依赖
pip install -r requirements.txt

# 配置 API 密钥（编辑 backend/config/settings.json）
# 启动服务
cd backend && python main.py
2

在你的网页中加一行代码

在 HTML 的任意位置（建议在 </body> 前）添加：

<script src="http://localhost:8001/voltaire.js"></script>

页面右下角会自动出现浮动按钮，点击即可打开 AI 对话框。无需任何额外配置。

3

（可选）使用管理后台完善知识库

打开 frontend/admin.html，通过可视化界面上传文档、管理 Agent 提示词、生成站点地图

试试看：打开项目根目录的 test.html，点击右下角浮动按钮，输入"帮我回到页面顶部" 或 "这个页面是做什么的？"

Knowledge Base

知识库系统

三类知识库协同工作，为 AI 提供完整的页面认知

🗺️

站点地图 (sitemap)

页面可交互元素库。存储按钮、链接、输入框等元素的 CSS 选择器、位置和可用操作。AI 生成操作代码时用于元素定位。由 DOM 自动扫描或 Playwright 工具填充。

📋

操作指引 (table)

结构化的操作步骤序列。例如"如何创建订单"的完整流程。引导 LLM 生成正确的分步操作，确保不遗漏关键步骤。通过文档上传或手动录入。

📖

文档资料 (document)

网站帮助文档、FAQ、产品说明等文本资料。用于回答知识性问题。上传时可选择 11 种优化策略（层次化索引、语义分块、知识图谱等）提升检索质量。

查询流程

1

跨类别联合查询

每次用户请求同时查询 sitemap、table、document 三类知识库，每类公平分配配额

2

LLM 语义筛选

从各库获取候选结果后，LLM 根据语义相关性精选 Top-K。操作类请求优先选 table > sitemap > document

3

注入生成上下文

精选后的元素和知识作为上下文注入到代码生成/问答的提示词中

文档优化策略

上传文档时可选优化策略，LLM 子 Agent 对内容进行智能处理：

策略 ID名称原理
none不优化（默认）按段落切分，零延迟
hierarchical_indices层次化索引先生成摘要层，再生成细节层，层次化检索
fusion_hybrid_search融合检索语义搜索 + 关键词搜索双路融合
cragCRAG评估检索质量，低相关度自动补充
graphrag知识图谱构建实体-关系图，结构化检索
document_augmentation文档增强预生成相关问题增加检索入口
共 11 种策略，详见 backend/knowledge/strategies.py
Agent System

Agent 配置

所有 AI 行为由 config/agents.json 驱动，管理后台可视化管理，修改即时生效

🎯

intent_classifier

意图分类器。分析用户消息，只返回一个单词：operation 或 question。

🤖

operation_agent

网站操作导师。将用户请求拆分为多步骤 JS 代码，每个步骤独立可执行，带教学说明。

💬

qa_agent

知识问答助手。基于知识库内容准确回答问题，知识库无匹配时诚实告知。

Agent 配置结构

{
  "id": "operation_agent",
  "type": "operation",          // operation | question | classifier
  "role": "你是网页自动化操作导师...",  // 系统角色设定
  "rules": [                         // 行为规则列表
    "规则1: ...",
    "规则2: ..."
  ],
  "tools": ["knowledge_query"],      // 可用工具
  "output_format": { ... }           // 输出格式模板
}
提示：所有 Agent 的 role 和 rules 决定了 LLM 的行为。通过管理后台或直接编辑 agents.json 即可调整——无需修改 Python 代码。

后端部署

开发环境部署 
# 1. 克隆项目
git clone <repo-url> && cd voltaire-ai

# 2. 创建虚拟环境
python -m venv venv && source venv/bin/activate  # Linux/Mac
# 或 venv\Scripts\activate  (Windows)

# 3. 安装依赖
pip install -r requirements.txt

# 4. 配置 LLM API
# 编辑 backend/config/settings.json，填入 API 密钥

# 5. 启动
cd backend && python main.py
# 服务运行在 http://localhost:8001
生产环境部署 
# 使用 uvicorn 多 worker 模式
uvicorn main:app --host 0.0.0.0 --port 8001 --workers 4

# 推荐: Nginx 反向代理
# /etc/nginx/sites-available/voltaire-ai:
server {
    listen 80;
    server_name ai.your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:8001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    location /voltaire.js {
        alias /path/to/frontend/voltaire.js;
        expires 1h;
        add_header Cache-Control "public";
    }
}

# systemd 服务 (推荐)
# /etc/systemd/system/voltaire-ai.service:
[Unit]
Description=VoltaireAI Backend
After=network.target

[Service]
User=www-data
WorkingDirectory=/opt/voltaire-ai/backend
ExecStart=/opt/voltaire-ai/venv/bin/uvicorn main:app \
  --host 127.0.0.1 --port 8001 --workers 4
Restart=always

[Install]
WantedBy=multi-user.target
Docker 部署（推荐）

FROM python:3.11-slim

WORKDIR /app

# 构建与运行

  -v $(pwd)/backend/config:/app/backend/config \
  -v $(pwd)/backend/.voltaire_knowledge:/app/backend/.voltaire_knowledge \
  --name voltaire-ai voltaire-ai
LLM 提供商配置

支持任何兼容 OpenAI API 的服务。在 settings.json 中添加即可：

// DeepSeek
{ "name": "deepseek", "base_url": "https://api.deepseek.com", "model": "deepseek-chat" }

// OpenAI
{ "name": "openai", "base_url": "https://api.openai.com/v1", "model": "gpt-4o" }

// 本地 Ollama
{ "name": "ollama", "base_url": "http://localhost:11434/v1", "model": "qwen2.5" }

// 自定义 vLLM / 兼容服务
{ "name": "custom", "base_url": "http://your-server/v1", "model": "your-model" }

设置 "default_model_source" 选择默认使用的源。

关键设计决策

决策说明
LLM 驱动一切意图分类、元素筛选、代码生成、知识库优化全部由 LLM 完成，无硬编码规则
配置驱动 Agent所有提示词在 agents.json 管理，修改后自动热加载
双模存储优先 ChromaDB 向量搜索，不可用时 JSON + LLM 关键词分解
SSE 替代 WebSocketServer-Sent Events 实现流式推送，防火墙友好
单文件前端voltaire.js 自包含所有样式和逻辑，一行 script 集成
失败自修复代码执行失败 → 错误反馈 LLM → 重新生成 → 前端重试
